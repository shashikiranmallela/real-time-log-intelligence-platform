# Day 1.4 — Kafka Consumer Service

## Objectives

- Build Kafka consumer service
- Read real-time streaming logs
- Process structured JSON log events
- Prepare backend pipeline for anomaly detection

---

## Kafka Consumer

The consumer subscribes to the `application-logs` Kafka topic and continuously processes incoming log events.

---

## Features

- Real-time log consumption
- JSON deserialization
- Continuous stream processing
- Consumer group support

---

## Notes

The consumer service forms the foundation for future anomaly detection, alerting, and log intelligence processing.
