# Day 1.5 — Real-Time Anomaly Detection

## Objectives

- Detect abnormal system behavior
- Implement threshold-based detection
- Implement error spike detection
- Implement statistical anomaly detection using Z-score

---

## Detection Methods

### Threshold Detection
Detects high response times above 4000ms.

### Error Spike Detection
Tracks repeated ERROR logs per service.

### Z-Score Detection
Uses statistical analysis to identify abnormal response times.

---

## Notes

This phase introduces intelligence into the streaming pipeline by identifying operational anomalies in real time.
